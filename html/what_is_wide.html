What is Wide ?

A package named Y2018.Text.Wide and it's subpackages specifies in Ada-GNAT what "Wide" is. But more important is the extension to Ada-GNAT text handling capabilities. The idea is that "String", "Wide_Wide_String", "Unbounded_String", "Unbounded_Wide_Wide_String", "Character" and "Wide_Wide_Character" are converted under the hood. When needed in a string-expression conversion between UTF-8 and UTF-32 is done. Look at row 12 with "Put_Line". Instead of using &-operator we are using an overloaded +-operator.
Look at rows 5 and 10, A special string constant with the syntax "..."W as a wide_wide_string constant.
01| procedure Sample is
02|  EXODUS:constant String:="Och detta är namnen på ";
03|  p:Pattern_TY;
04| begin
05|  Pattern(p,"^.*?(I[\w]+).*?([\w]+)"W);
06|  Put_Line ("--- Begin of Sample ---");
07|  declare
08|   r:Unbounded_Wide_Wide_String_AA(0 .. (size(p) - 1));
09|  begin
10|   if matchTo(EXODUS + "Israels söner"W,p,r) then
11|    for i in r'First .. r'Last loop
12|     Put_Line (">" + Integer'Image(i) + "=" + r(i) + "<");
13|    end loop;
14|   end if;
15|  end;
16|  Put_Line ("*** End of Sample ***");
17| end Sample;
Ada-GNAT uses unicode for character coding but unicode has five different character storage methods (UTF-8, UTF-16BE, UTF-16LE, UTF-32BE and UTF-32LE). In Ada-GNAT little endian is not supported. The Wide solution supports UTF-8 and UTF-32BE as two different character types with corresponding constant notation.
By not using &-operator and replace this with then overloaded +-operator it is possible to mix unbounded and fixed strings in the same expression.
Assigning a character variable can not be overloaded, but the expression can be converted to the correct type by 'adding' a null length constant at the beginning, one of "", ""W, NUS (constant Null_Unbounded_String) or NUW (constant Null_Unbounded_Wide_Wide_String).

The Wide solution supports Regular Expression:
     
01| with Ada.Text_IO; use Ada.Text_IO;
02| with Ada.Strings.Unbounded; use Ada.Strings.Unbounded;
03| with Ada.Strings.Wide_Wide_Unbounded; use Ada.Strings.Wide_Wide_Unbounded;
04| with Y2018.Text.Wide; use Y2018.Text.Wide;
05| with Y2018.Text.Wide.STR; use Y2018.Text.Wide.STR;
06| with Y2018.Text.Wide.Tool; use Y2018.Text.Wide.Tool;
07| with Y2018.Text.Regex; use Y2018.Text.Regex;
08| with Y2018.Text.Regex.PatternPack;
09| -- with DSECT;
10| procedure Star is
11|  pstring:Wide_Wide_String:="(..)(.*)"W;
12|  p:PatternPack.Pattern_TY;
13| begin
14|  Y2018.Text.Regex.PatternPack.Pattern(p,pstring);
15|  declare
16|   r:Unbounded_Wide_Wide_String_AA(0 .. (PatternPack.size(p) - 1));
17|  begin
18|   if Tool.matchTo("AZsome text",p,r) then
19|    for i in r'First .. r'Last loop
20|     Ada.Text_IO.Put_Line ("[" + Integer'Image(i) + "=" + r(i) + "]");
21|    end loop;
22|   end if;
23|  end;
24| end Star;
Lines 1 .. 8 with-statements.
Line 9 a special comment for the preprocessor.
Line 11 defines a Wide_Wide_String constant specifying how the regular expression pattern should be built
Line 12 a varible of pattern type
Line 14 construction of the pattern into variable p. The pattern is three structure.
Line 16 an unbounded wide wide string array with a dimension depending on number of capture expressions in the pattern and plus one, for the whole found expression. Indexing as in Perl language, index zero for all, and index from one uppward for capture expressions
Line 18 match the text "AZsome text" to the pattern and return the result into the r-variable when a match is found.
Line 20 Show the result
and when the procedure star is running we get following result
01| [ 0=AZsome text]
02| [ 1=AZ]
03| [ 2=some text]
 resultline 1 : the expression
resultline 2 : first capture expression
resultline 3 : second capture expression

'with' statements of Y2018.Text.Wide

The package Y2018.Text.Wide cannot be translated, but must first be preprocessed with y2018text processor.

  • Y2018.Text.Wide [04] - (contains constants NUS and NUW)
  • Y2018.Text.Wide.STR [05] - "+" operator overloading. If overloaded +-operator is not used then no with-statement is needed. This solution makes it possible to overload +-operator differently.
  • Y2018.Text.Wide.Tool [06] - tools for string processing
  • Y2018.Text.Wide.UTF - conversion between UTF-8 and UTF-32 storage methods
Y2018.Text.Wide is responsible for the smooth casting between different string types, and casting to specific string-type for the result can be made with prefixing it with a null-constant and the +-operator
  • "" + :cast to fixed String
  • ""W + :cast to fixed Wide_Wide_String
  • NUS + :cast to Unbounded_String
  • NUW + :cast to Unbounded_Wide_Wide_String
In Y2018.Text.Wide.Tool support for all four string-types exists.
