Y2018XMLflt

Y2018XMLflt, retrieve from a XML-file tags, comments, data. Y2018XMLflt should be used as a filter in a (shell-) command. Input is from standard in (STDIN) and output is to standard out (STDOUT).

Initiate

  • Is project Y2018-Text "maked" ? There should be a directory for the project and the makefile of the project has been executed. Lets call this directory "vp" ($vp) on this page.
  • Create a new directory for Y2018XMLflt. Lets call it "xmlfltdir" directory ($xmlfltdir).
  • xmlflt.tar in tar archive and corresponding SHA-256. Download and verify.
  • Explode xmlflt.tar file in xmlfltdir (cd xmlfltdir;tar x -f ../xmlflt.tar)
  • Create a sub-directory to xmlfltdir, call this sub-directory "tmp" (xmlflt/tmp directory, $tmp)
After exploding xmlflt.tar the directory contains html.properties and y2018xmlflt.adb files. File html.properties has properties for HTML data. File y2018xmlflt.adb is source code which should be compiled by y2018text and gnatmake.

Compile

Compile Y2018XMLflt.adb with command
export LD_LIBRARY_PATH=$vp/lib;$vp/bin/y2018text $xmlfltdir/Y2018XMLflt.adb $tmp
Note directories are specified as variables ($vp, $xmlflt and $tmp), and correct values must be substituted before execution

After successful preprocessor compilation (y2018text compilation) the result in tmp must be compiled with gnatmake :
cd $tmp;gnatmake -aI$vp/ads -aO$vp/ali Y2018XMLflt.adb -largs -ly2018text -L$vp/lib
Note again that directories are specified as variables ($tmp and $vp)

In tmp directory after a successful gnatmake compilation the executable is here named 'y2018xmlflt'.

Test run ?
Execute './y2018xmlflt -help' in tmp directory ($tmp). The executable will print a help on standard error (STDERR).

Run

Syntax: y2018xmlflt {options}* <{input-file} >{output-file}
Standard error (STDERR) is used for error and info messages.
Example: $tmp/y2018xmlflt $xmlfltdir/html.properties <y2018xmlflt.html >y2018xmlflt.txt

Options

Three types of options supported
  1. properties files, a file with the suffix .prop or .properties
  2. Syntax: -{name}
  3. Syntax: -{name}:{value}
Where {name} is a word and {value} is a string without blanks.
A property is generated from the content of a properties file or from a option. Once generated it cannot change.

Input and Output

Input is a XML file or maybe a HTML file. Output is a text file where every line has following syntax:
  • "P_ty" value or continue marker ('...') (see P_ty values)
  • zero or more parameters separated by space
  • '|' character
  • value depending on "P_ty" value
The usage is to programmatically get information from a XML document in a 'properties' like form.

Properties

Simple properties
-script
 0 or 1. If "1" do not analyze contents of a script tag. <script> tag contains code and this code may interfer with the analyzing process. Value "1" change to process to only seek </script> tag in the input stream
-linecount
 modulo value for message generating when counting input lines. Reason for this property is when the inputdata contains a huge amount of lines then the process seems to be not working but by setting this value to 1000 or something else we get a linecount message to standard error  
-width
 linewidth at output. A possibility to limit the line length in output
-clip
 maximi input line width. Defaults to 1000. To limit input lines by splitting.
-l
 list all properties on standard error
-help
 print on standard error a short help and exit from the program

Properties for substitution:
Syntax: &{val1}; "{character}" <<  -- any string in the input with value &{val1}; is in output substituted with {character}.

Special properties:
beg.attr=Y -- publish begin tag attribute values
tag.{tagname}=short -- a begin tag without an corresponding end tag

Substitution and special properties can only be specified in a properties file.

XML provides a mechanism to indicate that particular pieces of the document should be processed in a special way. These are called “marked sections”. Marked Section has following types or keywords
  • IGNORE
  • INCLUDE
  • CDATA
  • RCDATA
  • TEMP
Syntax for a marked section is
<![keyword[ content ]]>
where keyword is one of IGNORE, INCLUDE, CDATA, RCDATA or TEMP. A marked section ends allways with "]]>".

NOTE: Special P_ty value "..." is used for continuation lines in output.

Values before '|'-character and after P_ty value.
  • P_DA* -- data
  • P_XML -- xml-tag
  • P_DOC -- doc-tag
  • P_COM* -- comment
  • P_MS* -- Marked section. No substitution is done
  • P_END* -- end-tag: P_END1 name
  • P_SHORT* -- short-tag: SHORT0 tag-name, SHORT1 value inside tag
  • P_BEG* -- begin-tag: BEG0 tag-name, BEG1 tag-name depth, BEG3 tag-name, BEG4 tag-name
  • P_ATTR -- attribute: tag-name'attribute-name
  • ... -- continuation data
