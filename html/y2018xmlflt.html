XMLfilter

XMLfilter, retrieve from a XML-file tags, comments, data. XMLfilter should be used as a filter in a (shell-) command. Input is from standard in (STDIN) and output is to standard out (STDOUT).

  • Explode xmlflt.tar file in xmlfltdir (cd xmlfltdir;tar x -f ../xmlflt.tar)

Make

Make utility can be used after we modify 'makefile_proto.txt' and save it under the name 'makefile'.
The modification means replacing text '<DIR>' with the path of "vp".
Write after makefile is created "make" on the console.

Run

Write on the console "make run".

From the make file we get the command (note instead of <DIR> there should be "vp" path)
    export LD_LIBRARY_PATH="<DIR>lib";./xmlfilter html.properties <test_in.html
Output is routed to STDOUT (Standard out) but this is merged with linecounter and message data from STDERR (Standard error) (The solution here is only for test purposes).

Options

Three types of options supported
  1. properties files, a file with the suffix .prop or .properties
  2. Syntax: -{name}
  3. Syntax: -{name}:{value}
Where {name} is a word and {value} is a string without blanks.
A property is generated from the content of a properties file or from a option. Once generated it cannot change.

Input and Output

Input is a XML file or maybe a HTML file. Output is a text file where every line has following syntax:
  • "P_ty" value or continue marker ('...') (see P_ty values)
  • zero or more parameters separated by space
  • '|' character
  • value depending on "P_ty" value
The usage is to programmatically get information from a XML document in a 'properties' like form.

Properties

Simple properties
-script
 0 or 1. If "1" do not analyze contents of a script tag. <script> tag contains code and this code may interfer with the analyzing process. Value "1" change to process to only seek </script> tag in the input stream
-width
 linewidth at output. A possibility to limit the line length in output
-clip
 maximi input line width. Defaults to 1000. To limit input lines by splitting.
-l
 list all properties on standard error
-help
 print on standard error a short help and exit from the program

Properties for substitution:
Syntax: &{val1}; "{character}" <<  -- any string in the input with value &{val1}; is in output substituted with {character}.

Special properties:
beg.attr=Y -- publish begin tag attribute values
tag.{tagname}=short -- a begin tag without an corresponding end tag

Substitution and special properties can only be specified in a properties file.

XML provides a mechanism to indicate that particular pieces of the document should be processed in a special way. These are called “marked sections”. Marked Section has following types or keywords
  • IGNORE
  • INCLUDE
  • CDATA
  • RCDATA
  • TEMP
Syntax for a marked section is
<![keyword[ content ]]>
where keyword is one of IGNORE, INCLUDE, CDATA, RCDATA or TEMP. A marked section ends allways with "]]>".

NOTE: Special P_ty value "..." is used for continuation lines in output.

Values before '|'-character and after P_ty value.
  • P_DA* -- data
  • P_XML -- xml-tag
  • P_DOC -- doc-tag
  • P_COM* -- comment
  • P_MS* -- Marked section. No substitution is done
  • P_END* -- end-tag: P_END1 name
  • P_SHORT* -- short-tag: SHORT0 tag-name, SHORT1 value inside tag
  • P_BEG* -- begin-tag: BEG0 tag-name, BEG1 tag-name depth, BEG3 tag-name, BEG4 tag-name
  • P_ATTR -- attribute: tag-name'attribute-name
  • ... -- continuation data
