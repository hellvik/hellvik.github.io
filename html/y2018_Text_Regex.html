Implementation of Regular Expression in Y2018.Text

Some specifics about this implementation
  •  the solution is based on wide_wide_string (32 bit). To make it nicely function the preprocessor is needed. All references to hexadecimal values in standard regular expression is removed and the preprocessor is used to generate these characters
  • the notation of 'new line' is not supported. The reason is that 'new line' varies depending on the platform and OS
  • this is a implementation for Unicode UTF32 character base, all references to ASCII features are replaced  with UTF32

  2. No regular expression support, use preprocessor hexadecimal notation

  4. Use a user defined group

  7. Y2018.Text do not support line breaks
  8. Future consideration

Split ?

Split function do not exists but you can write it. Here as an example Splity :


with Ada.Strings.Wide_Wide_Unbounded; use Ada.Strings.Wide_Wide_Unbounded;
with Ada.Text_IO;
with Y2018.Text.Wide; use Y2018.Text.Wide;
with Y2018.Text.Wide.STR; use Y2018.Text.Wide.STR;
with Y2018.Text.Wide.Tool; use Y2018.Text.Wide.Tool;
with Y2018.Text.Regex.PatternPack;
with Y2018.Text.Regex.ToilPack;

procedure Splity is
    myPattern:constant Wide_Wide_String:="^(.*?),[\\s]*"W;
    p:Y2018.Text.Regex.PatternPack.Pattern_TY;
    myData:Unbounded_Wide_Wide_String:=NUW + "Huey, Dewey, Louie, Duck"W;
    v:Y2018.Text.Regex.G_Vector.Vector;
    i:Integer:=0;
    c:Y2018.Text.Regex.G_Vector.Cursor;
    j:Y2018.Text.Regex.G_Element;
begin
    Y2018.Text.Regex.PatternPack.Pattern(p,myPattern);
    loop
        declare
            t:Y2018.Text.Regex.ToilPack.Toil_TY; -- it is important to get a new instance of 't'
        begin
            Y2018.Text.Regex.PatternPack.Toil(t,p,""W + myData);
            if Y2018.Text.Regex.ToilPack.has_toil(t) then
                v:=Y2018.Text.Regex.ToilPack.Match(t);
                i:=0;
                for c in Y2018.Text.Regex.G_Vector.Iterate(v) loop
                    j:=Y2018.Text.Regex.G_Vector.Element(c);
                    if j.exists then
                        if i>0 then -- i=0 is always then orginal input, and we do not care about it
                            Ada.Text_IO.Put_Line ("> " + j.v);
                        end if;
                    else
                        if i>0 then
                            Ada.Text_IO.Put_Line ("> NO VALUE");
                        end if;
                    end if;
                    i:=i+1;
                end loop;
                myData:=NUW + Y2018.Text.Regex.ToilPack.Tail(t);
                Ada.Text_IO.Put_Line ("Tail" + "=" + myData); -- for tracing only
            else
                exit;
            end if;
        end;
    end loop;
   Ada.Text_IO.Put_Line ("*** End of Splity ***");
end Splity;

The reason why we do not have a "split" in Y2018.Text is that dynamic tables in Ada are vectors and, as you can see above, not so easy to handle.
