Y2018Chess

Utility for listing an UTF-8 file as a file dump where every line has a fixed amount of (unicode) characters and every character is of equal size. Control characters, undefined characters or illegal characters are replaced with a chess symbol.
вҷ”   WHITE KING (null) '00'
вҷ•   WHITE QUEEN (backspace) '08'x
вҷ–   WHITE ROOK (form feed) '0C'x
вҷ—   WHITE BISHOP (line feed) '0A'x
вҷҳ   WHITE KNIGHT (horizontal tabulation) '09'x
вҷҷ   WHITE PAWN (carriage return) '0D'x
вҷҹ   BLACK PAWN (next line) '85'x
вҷҡ   BLACK KING (control) Unicode general category Cc
вҷӣ   BLACK QUEEN (format) Unicode general category Cf
вҷң   BLACK ROOK (surrogate) Unicode general category Cs
вҷқ   BLACK BISHOP (not assigned) Unicode general category Cn
вҷһ  BLACK KNIGHT (illegal byte)
пҝҪ  Unicode replacement character

On Linux platforms the utlity name is y2018chess, the utility is depending on lib/liby2018text.so. The full command will be:
export LD_LIBRARY_PATH="<>/lib";<>/bin/y2018chess
where <> is the name of the path where the tar file has been exploded and where 'make' was given.

By only giving the utlity name as a command we get
  All control characters are replaced by CHESS symbols.
  Line length is fixed to the value of -blk parameter (default 50),
  'line length' is the number of UTF-8 characters on a single line.
  All non-UTF8 characters are replaced by BLACK KNIGHT symbol.
  вҷ”   вҷ•   вҷ–   вҷ—   вҷҳ   вҷҷ   вҷҹ   вҷҡ   вҷӣ   вҷң   вҷқ   вҷһ
  Syntax:
    Y2018Chess {<option>}* <input_file> {<output_file>}
  if no <output_file> then name of <output_file> is chess.lst
  Options:
    -blk:<line length> -- length of line in output
    -g -- use greek characters not chess symbols
    -r -- replace characters with a code value over 16#FFFF#
on the command screen.
About the options
  • -blk, the row length. Depending on the editor you are using a value of 50 is probably too small
  • -g, an alternate solution if chess symbols cannot be used
  • -r, to solve problems with output for tools which do not recognize UTF-8 values greater than 16#FFFF#. Theese values are legal UTF-8 values but...
