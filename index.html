ADA programming language project Y2018-Text

To make text handling workable in a ADA programming language and to introduce Regular Expression based on Unicode 12. The solution extends native ADA with Wide_Wide_String literals with backslash notation for BS, FF, LF, CR and HT control characters and hexadecimal values. &-operator is replaced with +-operator to make it possible to overload concatenations of string expressions, &-operator works as specified in ADA standard. Unicode.org general character categories and subcategories are supported, and Regular Expression implementation is extended with this.
